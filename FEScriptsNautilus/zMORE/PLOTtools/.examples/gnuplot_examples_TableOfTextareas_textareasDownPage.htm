INTRODUCTION :

Unfortunately, the books and user manuals on 'gnuplot' tend to present lots of 'snippets' of commands --- but not many complete command examples.

This guide is intended to fill that void by presenting ONLY complete working examples. Those that require an input data set will have the dataset presented toward the bottom of this guide.

Since there are many optional parameters available for gnuplot commands, including

  • line widths
  • line styles
  • fonts
  • etc.

there will be comments above or below many of the command lines in these examples, to explain some of those options. The comments will (hopefully) provide a clear idea of some of the more useful options available to the user.

You can navigate to the gnuplot-script examples by using the following Table of Contents.

OR, you can use the 'Find Text' option of your web browser to find keywords in the text in this page (including the code in the gnuplot-scripts) --- keywords such as 'label', 'title', 'xtic', 'legend', 'timestamp', etc.

How the code samples (below) are presented - and other notes :

Each code 'sample' is presented via an HTML 'textarea' so that you can scroll the text, vertically or horizontally --- if the text is many-lined or wide.

There are a lot of comments in the scripts. The comment lines start with a '#' (hash) sign. Generally, there will be a single hash sign on lines that contain valid commands that could be executed if de-commented. A double-hash indicates a line that should never be de-commented, because it is not executable gnuplot command, rather it is text for information only.

Use the table-of-contents (above) to go directly to code samples. OR simply scroll down this page to spot samples of interest.

Alternatively, use the 'Find text' option of your web browser to look for keywords on this page.

The code is presented here mostly for my own personal use --- as a reference and as a means of archival and backup. I may use portions of the scripts for working code fragments to be applied in other scripts. If others use these scripts, they should test them on test data before using in a 'serious' mode.

Since these scripts do not change any files, they should not cause any harm other than creating some small plot image files that take up some disk space.

However, when anyone uses these scripts, they are hereby notified that they do so at their own risk.
